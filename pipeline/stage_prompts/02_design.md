# Stage 2: DESIGN

> 仅 full 路由（4+文件变更或新架构）需要此阶段。其他路由跳过。

## 你现在只做一件事：设计架构，让实现阶段无需做决策

不写实现代码。确定数据怎么存、接口怎么定、模块怎么分，让 Stage 3 的 Sonnet 只需要"翻译成代码"。

### 输入

- Stage 1 的验收标准和影响文件清单
- 项目现有代码结构

### 你的任务

1. **设计数据模型**：新增/修改哪些表或数据结构
2. **定义接口契约**：函数签名、参数类型、返回值类型
3. **划分模块边界**：哪些代码放哪个文件，模块间通过什么接口通信
4. **确定实现顺序**：哪个模块先实现（被依赖的先）

### 输出

保存到 `.harness/design.md`，格式：

```
## 数据模型
[表结构/数据结构定义]

## 接口契约
[函数签名 + docstring，不写实现]

## 模块划分
[模块职责 + 依赖关系]

## 实现顺序
1. 模块X（无依赖，先实现）
2. 模块Y（依赖X）
```

### 验证条件

- 每个 Stage 1 影响文件都在模块划分中出现
- 模块间无循环依赖
- 每个接口有明确的输入输出类型

### 绝对不做

- **不写实现代码**：函数体不写逻辑，只写签名
- **不写测试**
- **不做性能优化设计**
